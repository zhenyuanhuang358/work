Phase 4 使用。三层信号汇总 → 一句话判断 → 标的替换清单 → （可选）HTML 报告。

---

## 二、因果链解读（必填，不能只列表格）

```
## 因果链解读

**第一层触发时间**：[描述何时出现扩张决策信号，具体到季度]

**第二层传导时间**：[描述何时出现门店质量信号，与第一层的时间差]

**第三层确认时间**：[描述财务指标何时开始反映，或尚未反映]

**反直觉信号说明**：
- [差评结构变化的真实含义，不只是描述现象]
- [等位/翻台率变化的真实含义]

**当前因果链所处位置**：
□ 仅第一层信号（最早期，T-2）
□ 第一层 + 第二层（扩张失序已传导至门店，T-1）
□ 三层全亮（财报即将反映，T-0）
□ 第三层已确认，前两层回看验证
```

## 五、HTML 报告（可选，用户要求时生成）

文件路径：`reports/[slug]_expansion_[YYYY].html`

slug 规则：品牌名拼音缩写，小写，下划线连接
示例：`haidilao_expansion_2025.html`、`banu_expansion_2025.html`

### HTML 视觉规范

遵循 restaurant-research-skill 的报告视觉标准：

```css
:root {
  --ink:#0a0a0b; --paper:#f1efea; --copper:#8b6c42; --gold:#c9a84c;
  --green:#2a5c3f; --red:#8b2e2e; --amber:#c47a1e;
  --muted:#6b6460; --border:#d4cec6; --card:#faf8f4;
}
字体：'IM Fell English'（英文）+ 'Songti SC','STSong','SimSun',Georgia,serif（中文）
```

### HTML Section 顺序

1. **Pub Bar**：品牌名 · 扩张健康度诊断 · 日期
2. **双语标题**：中文大标 + 英文副标 + 标签行（上市状态/品类/客单价）
3. **Stat Bar**：4格关键数字（如：总门店数 / 年净增率 / 翻台率 / 新老店剪刀差）
4. **三层信号汇总表**（含 ✅/⚠️/🔴 徽章）
5. **因果链 SVG 时间轴**：三层信号触发时间点可视化
6. **扩张速率折线图**（SVG，月均净增曲线，标注失序阈值线）
7. **新老店评分对比图**（SVG，柱状图或折线）
8. **翻台率趋势图**（SVG，一线 vs 三线分化）
9. **一句话判断框**（深色底，定价缺口方向 + 时间预估）
10. **数据缺口补全建议表**（如有缺口）
11. **来源索引**
12. **Footer**

### SVG 图表必须包含

| 图表 | 类型 | 内容 |
|------|------|------|
| 扩张速率曲线 | 折线图 | 月均净增门店数，标注历史均值线和失序阈值线 |
| 城市层级分布变化 | 堆叠柱状图 | 每年一线/二线/三线占比 |
| 新老店评分对比 | 柱状图 | 近2年新店 vs 5年以上老店，按季度 |
| 翻台率区域截面 | 折线图（如有数据） | 整体、一线、三线三条线 |

## 六、质量自检（输出前必须核对）

```
□ 三层都有实际数据填入（不是占位符或"数据缺口"未说明替代方案）？
□ 反直觉信号解读已应用于第二层（差评结构解读的是真实原因不是表面现象）？
□ 一句话判断有明确方向（风险/机会/中性）和时间预估？
□ 所有数据标注了来源和日期？
□ baseline-library.md 的失序期参数已被引用作比较基准？
□ 客单价对应的蚕食半径档位已填入？
□ 因果链时间顺序正确（第一层触发 > 第二层传导 > 第三层确认）？
□ "市场是否已定价"列有实质性判断（不是空填）？
```
