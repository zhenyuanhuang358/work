# 所有者盈余模型与内在价值计算

---

## 核心公式

### Owner Earnings（所有者盈余）

```
Owner Earnings = 净利润
              + 折旧摊销（D&A）
              - 维护性资本支出（Maintenance Capex）
              ± 运营资本变化（ΔWorking Capital，WC增加为负/WC减少为正）
```

**为什么不用净利润**：净利润包含非现金项目（折旧、摊销），且 Capex 不走利润表但消耗真实现金。

**OE < 0 的处理**：若 Owner Earnings 计算结果 < 0（正净利润但 Capex 需求超过净利润+D&A），DCF 同样不可靠——负 OE 意味着公司靠外部资本维持运营。此时参照「FCF 负值」处理路径，触发估值失效条件，在报告 §二 标注「OE 为负，DCF 不运行」，转替代估值。

**维护性 vs 增长性 Capex 的拆分**：
- 理想来源：公司在财报中主动披露（少数公司会做）
- 重资产行业（制造/地产/能源）：D&A 接近维护性 Capex，可用 D&A 作为代理
- 轻资产行业（软件/消费品/IP公司/无晶圆厂半导体 Fabless）：D&A 主要是无形资产摊销，**不能**代表维护性 Capex；改用总 Capex 的 40-60% 作为维护性部分，并在报告中注明假设
- 如无法拆分且行业不明：用总 Capex（最保守处理）

## DCF 两情景模型

### 基准年份选取（每次必做，优先于一切计算）

**使用距分析日期最近的完整财年**。判断规则：
- 若财年已结束 > 45 天且财报已发布 → 必须使用该财年数据
- 例：分析日期 2026-05-27，AMD 财年结束 12 月 → FY2025（已结束5个月）数据应已发布，必须用 FY2025
- 若仍使用更旧财年（如因数据采集限制），须在报告 §七「数据盲区」中注明，并说明对 OE 基准的影响方向

**影响**：使用过旧的 OE 基准会系统性低估或高估 IV，尤其对处于快速成长期的公司影响显著。

---

### 折现率（r）选取

| 市场 | 无风险利率基准 | 股权风险溢价 | 推荐折现率 |
|------|------------|-----------|---------|
| 港股大盘股 | 美债10年期 | +4-5% | 9-10% |
| 港股中小盘 | 美债10年期 | +5-7% | 11-12% |
| 美股大盘股 | 美债10年期 | +4% | 9% |
| A股 | 中国国债10年期 | +5-6% | 10-11% |

> **动态调整**：表中"推荐折现率"基于约4-5%无风险利率。每次分析前查当前10年期国债收益率，若偏离4-5%超过1%，按「当前无风险利率 + 对应风险溢价」重新计算。利率环境变化对 IV 影响显著，不要套用过期固定值。

**不调整折现率来「使结果合理」**——这是估值最常见的错误。折现率由市场风险决定，不由"想要的价格"决定。

---

### 阶段增速上限（来自 moat-framework.md 护城河评分）

| 护城河评分 | 阶段1最高增速上限 | 终端增长率上限 |
|-----------|----------------|-------------|
| 3-4（窄）  | 8%             | 3%          |
| 5-6（中等）| 12%            | 5%          |
| 7-8（宽）  | 18%            | 8%          |
| 9-10（卓越）| 25%           | 10%         |

> 硬规则：终端增长率 g_terminal 不得超过折现率 r - 2%（否则公式发散）。

**终端增长率参考值**（选最保守的适用值）：
- 纯国内业务（A股/港股国内）：用中国长期 GDP 预期（2-3.5%）
- 全球化/多市场公司：用全球 GDP 预期（2.5-3%）
- 美股本土公司：用美国长期 GDP（2-2.5%）

### ⚠️ 异常增速处理（近3年 FCF CAGR > 50%，或 FCF 近期首次由负转正时必读，优先于下方公式）

套用公式会产生荒谬结果（如 270% × 60% = 162%）。改用以下流程：

**⚠️ FCF 由负转正（CAGR 无意义）**：若 FCF 近期首次转正（分母为负），CAGR 计算无意义，直接跳至第2步，用分析师共识锚定增速；若无共识，跳至第3步用 FCF yield 反推。

**第1步：判断增速性质**
- 周期性爆发（大宗商品超级周期、换机周期）→ 用行业中周期均值替代历史 CAGR
- 结构性平台跃迁（如云计算 2014-2019、AI 加速器 2023-）→ 进入第2步

**第2步：用分析师共识锚定近期增速**
搜索：`[代码] revenue FCF growth estimate consensus 2026 2027 2028`
取未来3年共识预期均值作为 g₁ 参考，再与护城河上限取 min。

**第3步：若无可靠共识，使用 FCF yield 反推**
- 保守 g₁ = FCF yield（当前年度 FCF / 市值）+ 2%
- 基准 g₂ = FCF yield + 5%
- 两者均不得超过护城河阶段1上限

**第4步：报告中标注**："历史增速异常，DCF 基于正态化增速，可信度中等；
当前估值主要由市场对增速的预期驱动，安全边际计算意义有限。"

---

### 情景一：保守情景（Conservative Case）

```
阶段1增速 g₁ = min(近3年FCF复合增速 × 60%, 护城河对应阶段1上限)
阶段1：前5年以 g₁ 增长
阶段2增速 = g₁ × 50%，后5年
终端增长率 g_terminal = 对应上限的60%（不超过2.5%）

终值 = OE_year10 × (1 + g_terminal) / (r - g_terminal)
```

### 情景二：基准情景（Base Case）

```
阶段1增速 g₂ = min(近3年FCF复合增速 × 85%, 护城河对应阶段1上限)
阶段1：前5年以 g₂ 增长
阶段2增速 = g₂ × 60%，后5年
终端增长率 g_terminal = 对应上限的80%（不超过4%）
```

### 内在价值区间

```
保守 IV = PV(保守情景现金流) + PV(保守终值)
基准 IV = PV(基准情景现金流) + PV(基准终值)

最终内在价值区间 = [保守 IV, 基准 IV]
中值（报告使用）= (保守 + 基准) / 2
```

**⚠️ 终值集中度检验（每次必做）**

```
终值占比 = PV(终值) / 总 IV

若终值占比 > 60%：DCF 结果高度依赖终值假设，可信度下降。
必须用出口倍数法交叉验证：
  交叉验证 IV = Year10 OE × 同行当前 EV/FCF 中值 / (1+r)^10
若两种方法结果差距 > 30%，取较低值，并在报告中注明差异。
```

## 安全边际计算

```
安全边际 = (每股内在价值中值 - 当前股价) / 每股内在价值中值 × 100%

买入区间：安全边际 ≥ 30%（等于：当前价 ≤ 内在价值中值 × 70%）
观察区间：安全边际 15-29%
高估区间：安全边际 < 15%（不买，若已持有评估减仓）
```

## 三个关键检验

### 检验 1：FCF 转化率
```
FCF 转化率 = FCF / 净利润

正常：75-100%（FCF ≈ 净利润）
优秀：> 100%（FCF 持续超过净利润 = 高质量盈利）
警告：< 60%（连续3年）= 会计利润虚高，需检查 Capex 或应收账款
```

### 检验 2：ROIC vs 折现率
```
若 ROIC < r（折现率）：公司投入的每块钱在摧毁价值，增长越快越糟糕
若 ROIC > r × 1.5：高质量复利机器

ROIC = NOPAT / 投入资本
NOPAT = 营业利润 × (1 - 税率)
投入资本 = 股东权益 + 有息负债 - 现金
```

### 检验 3：资本分配质量（定性参考）

资本分配质量的**正式评分**在 `references/berkshire-score.md` §3A，此处仅列出检验要点：
- 回购时机：是否在估值偏低时（PE<20 或 PB<3）执行？
- 并购纪律：有无高溢价并购后商誉减值记录？
- 稀释控制：年度 SBC 稀释是否 < 2%？
- 债务纪律：有息负债增速是否低于 FCF 增速？

定性判断结果带入 berkshire-score.md §3A 的 base-6 计分框架，不在此处重复评分。

## 估值失效条件（放弃标准 DCF，转替代估值）

以下情况 DCF 数字不可信，必须在报告中声明：

1. FCF 连续3年为负（尚在投入期，终值主导估值 = 不可靠）
2. 营收增速波动率 > 50%（每年大幅变化，增长率无法合理外推）
3. 负债率 > 50% 且 EBIT/利息 < 3x（财务风险主导，非运营质量）
4. 行业处于根本性技术颠覆期（5年现金流不可预期）

### 替代估值顺序（满足条件后按顺序选用）

**第1选：EV/FCF 同行比较**（FCF 为正但其他条件触发时）
用同行 EV/FCF 中值 ± 20% 区间估算，注明「定性估值，可信度中等」。

**第2选：正态化 FCF 法**（FCF 连续为负、处于资本支出高峰期的公司）

适用条件：公司正进行大规模 Capex 建设（建厂 / 新平台），历史上曾有稳定正 FCF，且有明确盈利回归预期。

步骤：
1. 取公司历史正 FCF 峰值均值（峰值年及前后各1年，共3年均值）作为「正态化 FCF 基准」
2. 给出悲观 / 基准 / 乐观三情景目标 FCF，明确各情景成立条件
3. 目标 FCF × 行业合理倍数（15-22x，护城河评分 3-4 取 15x，5-6 取 18x，7+ 取 22x）= 目标 EV
   （倍数以「侵蚀调整后的护城河评分」为准；护城河处于侵蚀中时，倍数取下档）
4. 目标 EV - 净债务（或 + 净现金）= 目标权益价值；÷ 在外股数 = 每股 IV
5. 在报告 §四 中声明：「正态化 FCF 估算，可信度中等；价值兑现依赖具体转型假设」

**第3选：EV/EBITDA 或 EV/Sales**（无历史正 FCF，如初创期公司）
与同行倍数对比，声明「纯定性估值，可信度较低」。不得给出精确买入价。
