# Memory — 跨会话记忆中心

这是所有**跨会话、跨 skill** 记忆的唯一家。

一条原则：**记忆是数据，不是 skill**。skill 是给模型的上下文（指令/流程），
记忆是上下文之间沉淀下来的事实与状态。所以持仓、研究结论、历史教训都放这里，
不再埋进各 skill 的 `references/`。

---

## 读取约定（什么时候读哪个）

| 时机 | 读 |
|------|----|
| 每个会话第一次执行任务前 | `profile.md`（偏好/可复用规则/配置/教训，**必读**） |
| 期权扫描 / 持仓相关 | `state/options-journal.md` |
| 研究某公司前（Merlin / Earner / restaurant-research） | **先 grep `state/research-ledger.md`**，有沉淀就继承，不从零重做 |

## 写入约定（什么时候写什么）

| 触发 | 写 |
|------|----|
| 出现新事故 / 踩坑 / 用户纠正 | `profile.md` 第一节「执行经验」追加：根因 + 教训 + 规则 |
| 开仓 / 平仓 / 敞口变化 | `state/options-journal.md` 更新持仓表 |
| 交付一份研究报告 | `state/research-ledger.md` 追加该公司关键结论 + 置信度 + 报告路径 |

## 文件索引

| 文件 | 性质 | 内容 |
|------|------|------|
| `profile.md` | always-load · 慢变 | 偏好、可复用规则（R-*）、配置速查、历史教训 |
| `state/options-journal.md` | 动态 | 期权持仓、风险敞口、止损追踪 |
| `state/research-ledger.md` | 动态 · append-only | 研究结论沉淀，按公司索引 |

## 一条铁律

**写结论，不写过程。** 记忆里存的是「海底捞抖音综合费率约 2.5-3%（中置信，2026-06）」，
不是整段对话或整份报告。检索靠 grep，所以每条结论自带 **公司名 + 日期 + 置信度 + 报告路径**。

> 为什么不用向量库：单人规模下，结构化 markdown + grep 比向量检索更准、更可调试、零依赖。
> 等结论条目过千、需要语义检索时再考虑，现在是 premature optimization。
