## 一、窄门餐眼（第一层主力来源）

**用途**：门店列表、开业时间、城市、选址梯度、扩张速率

```
搜索方式：
直接访问：https://www.narrow-door.cn/brand/[品牌名]
或搜索：[品牌名] site:narrow-door.cn

可获取字段：
- 全国门店列表（城市/地址/开业时间/营业状态）
- 按年份统计新开/关闭门店数
- 城市层级分布（需手动对照分类标准）

如果窄门餐眼不可访问，备用搜索：
[品牌名] 全国门店数 城市分布 [年份]
[品牌名] 开店计划 新开门店 [年份] 
[品牌名] 门店扩张 一线 二线 三线
```

## 二、极海（第一层坐标计算）

**用途**：批量地址转坐标，计算门店间距离

```
搜索方式：
访问：https://geohey.com 或 极海地理数据平台

操作步骤：
1. 上传门店地址列表（CSV格式：品牌名/地址/开业时间）
2. 批量地理编码 → 获取经纬度
3. Python 计算距离矩阵（核心代码见下方）

距离计算 Python 代码片段：
from geopy.distance import geodesic
def min_distance(new_store_coord, existing_stores_coords):
    distances = [geodesic(new_store_coord, s).km for s in existing_stores_coords]
    return min(distances) if distances else None

备用方案（极海不可访问时）：
- 高德地图/百度地图开放 API（批量地理编码）
- 大众点评地图视图：切换"地图"模式，目测主要城市门店密度
  → 重点城市各做一个样本：北京/上海/成都/深圳
```

## 三、大众点评（第二层主力来源）

**用途**：门店评分、评论量、新老店爬坡曲线、差评分类

### 门店搜索操作

```
PC端：dianping.com → 搜索品牌名 → 切换"列表"视图
移动端：大众点评 App → 搜索 → 商家列表

筛选操作：
- 按"开业时间"排序（识别新老店）
- 按"评价数量"筛选（抽样时取评论量充足的门店）

评论数据采集：
1. 门店页 → "评价"tab → 按时间筛选
2. 每月抽取：总评分 + 口味/环境/服务各项评分 + 评论数量
3. 差评筛选：评分 2星及以下 → 按季度分批导出
```

### 差评 AI 分类指令

```
"以下是[品牌名]大众点评差评[N]条，日期范围[时间段]。
请将每条分类为：服务/口味/性价比/等位/其他。
输出格式：序号 | 原文（截取关键词） | 分类 | 核心词
最后输出各类别汇总数量和占比。"
```

### WebSearch 补充模板

```
[品牌名] 大众点评 评分 新店 开业 [城市] [年份]
[品牌名] 差评 服务 口味 性价比 [季度]
[品牌名] 大众点评 新店 vs 老店 评分 对比
```

## 六、数据质量标注规则

| 数据来源 | 标注方式 |
|---------|---------|
| 来自年报/公告（直接披露） | 【官方·年报 [年份]页[X]】 |
| 来自第三方数据库（窄门/大众点评） | 【平台数据·[来源]·采集日期】 |
| 来自媒体报道 | 【媒体报道·[来源]·日期】 |
| 用行业基准估算 | 【行业基准·估算值】 |
| 自行推算（如从总数倒推关闭数） | 【分析推算·注明方法】 |
| 无法获取 | 【数据缺口·建议补数路径】 |

**冲突处理规则**：
1. 官方一手 > 第三方平台 > 媒体报道 > 推算
2. 同来源新数据 > 旧数据
3. 两来源差距 > 15%：两个都标注，不做主观取舍，建议 IR 核实
