# 行业基准数据库

用于：① 非上市公司收入估算；② 分部数据推算；③ 验证已有数据合理性。
每个数字后标注来源和适用年份区间。所有估算结果必须标注`【分析预测·基准推算】`。

---

## 如何使用

**基本公式**：`收入估算 = 接待人次 × 人均消费（综合客单价）`

**综合客单价** = 门票 + 园内消费（餐饮+零售+游乐项目）+ 酒店住宿（按住宿渗透率折算）

**验证方法**：用已知竞品的 人次/收入 比率交叉验证。

## 四、分部数据估算方法

当公司不分部披露时，用以下方法拆分：

### 方法一：节假日数据推算法（主题公园）

```
步骤：
1. 收集该园节假日公告数据（春节/五一/国庆）
2. 节假日通常占全年人次 25–35%（旺季高度集中型乐园）
3. 全年估算 = 节假日总人次 / 节假日占比
4. 示例：郑州海昌五一16.5万人次，若五一占全年约3%
         → 全年约550万人次
```

### 方法二：竞争力得分占比法（⚠️ 仅在节假日公告数据不可得时使用）

```
步骤：
1. 查《主题公园竞争力评价报告》中各园得分
2. 同集团多园得分比例 ≈ 人次比例（粗估）
3. 用已知集团合计校验
```

> **注意**：竞争力评分体系考量品牌、设施等多维因素，与实际人次相关性较弱（误差通常 ±30%）。
> 报告中务必注明"粗估，仅供方向参考，误差较大"。优先使用方法一或方法三。

### 方法三：门店数/面积推算法（连锁餐饮）

```
步骤：
1. 确定区域/分部门店数占比
2. 乘以集团平均单店收入
3. 直营/加盟结构需分别计算（加盟只有抽成收入）
```

### 方法四：信息拼接组合推理法（无直接数据源时的默认方法，不允许留空）

> 适用场景：提纲问题公开渠道没有单一权威来源，但存在多个相邻、碎片化的真实信息可以组合推理。**这是默认应该尝试的方法，不是可选项**——只有连组合推理的锚点都完全找不到时，才允许标注纯数据缺口。

```
标准步骤：
1. 列出所有相关的真实锚点数据（哪怕只是相邻时期/相邻维度/同类事件）
2. 用显式假设把锚点连接成推理链条，每一步都写"锚点→假设→结论"，让链条可验证
3. 用行业基准/可比公司数据侧向校准推测范围（不是凭空猜，是用真实参照系框定合理区间）
4. 给区间估计，不给假精确的单点数字
5. 按假设层数标注置信度：1层假设（如线性外推已知速度）置信度中；3层以上假设（如层层推导的季节性拆分）置信度低，如实标注，不为了好看虚报
6. 结尾重申"非确认事实，正式使用前需一手核实"
```

**两个已验证的实战范式**（可直接复用套路）：

- **半年报/年报隐含拆分法**：已知 H1 实际值 + 全年实际值 → 隐含 H2 = 2×全年 − H1；已知全年值 + H1 内部占比假设(如春节旺季占比) → 拆出更细颗粒度估算。适用于任何"只有半年度/年度披露、但客户要更细颗粒度"的场景。
- **近期速度线性外推法**：已知两个时间点的门店数/规模数据 → 算出期间平均速度 → 外推到目标时点，给出区间而非单点。适用于"缺中间时点数据"的扩张类问题。

**反面案例（不要这样做）**：把"需要访谈本人"当作最终答案直接返回，或者把无法验证的数字当作确定事实呈现且不标注推测层级——这两种都不合格，前者是偷懒留白，后者是无标注地误导读者。
