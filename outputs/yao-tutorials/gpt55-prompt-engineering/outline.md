# 大纲：GPT-5.5 提示词工程——从入门到进阶

## 教程定位
- 受众：中文用户，会用ChatGPT，想系统提升提示词能力
- 目标：读完后能独立写出结构清晰、稳定可复用的提示词
- 长度：5000-10000字，7章

---

## 开篇（Hook）：同一个问题，两种截然不同的答案

**核心场景**：同样问GPT-5.5一道商业分析题，甲得到5行废话，乙得到带结构、有论据的深度报告。差距不在于谁用的模型更好，而在于提示词的质量。

**学习承诺**：本教程结束后，你将掌握一套可复用的提示词框架，以及针对GPT-5.5新能力的适配策略。

**章节内容**：

1.2 为什么提示词的表述方式会影响结果

**视觉**：Layer图——从用户文字到模型输出的处理流程

## 第2章：提示词的四要素结构——让每一句话都起作用

**学习目标**：掌握RTCF框架（角色、任务、上下文、格式），能写出完整结构的提示词

**章节内容**：
2.1 角色（Role）：告诉模型它是谁
2.2 任务（Task）：告诉模型做什么
2.3 上下文（Context）：给模型必要的背景信息
2.4 格式（Format）：告诉模型输出什么样的结果
2.5 RTCF组合实战：一个完整示例

**视觉**：Flow图——RTCF四要素流程图

## 实践任务

- 练习1（识别）：拿出你最近用过的一条提示词，用RTCF框架分析它缺少什么
- 练习2（改写）：将一条模糊指令改写为精确指令，并对比两次输出
- 练习3（应用）：为你工作中一个高频任务写一条带思维链的少样本提示词
