# 消费行业调研 · 工作流

> 输入客户提纲 → 识别品牌与格式 → 加载策略文件 → 数据采集 → Markdown报告 → **HTML报告（必做）**

---

## 核心原则

1. **提纲优先**：报告结构严格跟随客户提纲，不自作主张改变顺序或增减章节
2. **数据诚实**：找不到的数据明确标注`【数据缺口】`，不用模糊语言掩盖
3. **来源追溯**：每个关键数据点注明来源和日期
4. **分析补位**：数据缺口用`【分析预测】`行业判断补充并标注，不能留空

---

## 品牌分类与文件加载

收到提纲后先分类，然后读取对应策略文件再开始采集：

| 类型 | 判断标准 | 加载文件 |
|------|---------|---------|
| **A类：上市公司** | A股/港股/美股上市 | `references/A-listed.md` |
| **B类：非上市规模连锁** | 门店>100家但未上市 | `references/B-unlisted.md` |
| **C类：新兴品牌** | 门店<100家或成立<5年 | `references/C-emerging.md` |
| **专家访谈格式** | 用户明确要Q&A/访谈风格 | `references/expert-interview-format.md` |

**按需加载：**
- 不确定去哪找数据 / 触发 GitHub Action → `references/data-sources.md`
- 需要估算非上市收入 / 分部推算 / 验证数据合理性 → `references/industry-benchmarks.md`
- 指标定义有疑问 → `references/metric-definitions.md`
- 生成 HTML 报告（每次必做）→ `references/report-template.md`

## Phase 0：解析提纲 & 数据预备

### Step 0.0：检查本地缓存（上市公司必做）

对每个**上市公司**目标，先查 `research_cache/_manifest.json`：
- 若该 ticker 存在且 `updated_at` < 24小时 → 直接读 `research_cache/[slug].json`，跳到 Step 0.1
- 否则 → 用 `mcp__github__push_files` 推 `research_fetch_trigger.json` 触发 GitHub Action，轮询等待缓存刷新，再继续

```json
// research_fetch_trigger.json 格式
{"tickers": ["02255.HK", "9988.HK"], "triggered_at": "[ISO时间]", "triggered_by": "research-agent"}
```

### Step 0.1：提取基本信息

识别：目标品牌（名称+是否上市+品类）、调研时间段、核心问题（客户最关心的2-3个判断点）、提纲粒度（详细问题 vs 仅给方向）。

### Step 0.2：粒度判断

| 提纲类型 | 处理方式 |
|---------|---------|
| **详细型**（给出具体数据点） | 严格按提纲结构，补充所需数据 |
| **方向型**（只给品牌名+大方向） | 自动展开为8个标准维度 |
| **专家访谈型**（Q&A/访谈格式） | 加载 `references/expert-interview-format.md` |

**方向型默认展开8维度：**
经营规模概览 · 核心经营指标 · 扩张节奏与质量 · 盈利能力 · 战略动向 · 竞争格局 · 同比环比变化 · 预期与催化剂

### Step 0.3：解析确认

```
📋 提纲解析完成

目标品牌：XX（A类·港股）、YY（B类·非上市）
调研维度：[列出，若展开则说明"已按标准框架展开"]
重点关注：[客户最突出的2-3个判断点]
时间范围：[最近4个季度，或识别到的时间段]
报告格式：标准调研报告 / 专家访谈Q&A

确认无误后开始采集，或请指出需要调整的地方。
```

## Phase 2：数据结构化

### 核心数据表（每个品牌建一张）

| 指标 | 数值 | 时间 | 同比 | 来源 |
|------|------|------|------|------|
| 门店总数 | X家 | 2025Q4 | +Y% | 财报 |
| 营收 | X亿元 | 2025H2 | +Y% | 中报 |
| 净利润 | X亿元 | 2025H2 | +Y% | 中报 |
| 翻台率 | X次/天 | 2025H2 | ↓ | 中报 |
| 客单价 | X元 | 2025H2 | -Y% | 中报 |
| 【数据缺口】外卖占比 | — | — | — | 未公开披露 |

### 数据可靠性标注

- `【一手·财报】` 公司官方财务数据
- `【一手·公告】` 公司公告或IR材料
- `【二手·媒体】` 权威媒体（注明名称）
- `【二手·券商】` 券商报告（注明日期）
- `【推断】` 基于多方信息的推断
- `【分析预测】` 行业经验判断（数据缺口时必须补充）
- `【数据缺口】` 无法找到，需客户补充

## Phase 3：Markdown 报告

**严格按客户提纲章节排列。** 报告末尾附来源索引。

**执行摘要规则：**
- 格式：`1. [核心判断]（依据：[数据支撑，一句话]）`
- 禁止：「整体经营平稳」「市场竞争激烈」等无信息量表述
- 数据缺口项也必须给出分析判断，不留空白

**数据缺口处理：**

| 情况 | 处理方式 |
|------|---------|
| 上市公司未披露某指标 | `【未披露】` + `【分析预测】`行业推断 |
| 非上市公司财务数据 | `【非上市·数据不可得】` + 替代信号估算 |
| 新兴品牌多数指标 | 章节开头统一说明局限，不逐条标注 |
| 数据相互矛盾 | 并列呈现，标注来源，建议以哪个为准 |

**WebFetch 失败时降级：**

| 主源失败原因 | 降级路径 |
|------------|---------|
| 403 / 需登录 | 改 WebSearch，搜索同一页面的媒体转载摘要 |
| 页面无结构数据 | 搜索 `[品牌] 财报 [年份] 数据` 找媒体报道数字 |
| 非上市无财报 | 直接进入【分析预测】，标注估算逻辑 |

---

## Phase 4：HTML 报告（每次必做，不可跳过）

> 读取 `references/report-template.md`，按规范生成 HTML 文件。

1. 读取 `references/report-template.md` 确认规范
2. 按固定 Section 顺序构建 HTML
3. 为所有趋势/拆分数据生成 SVG 图表（不得纯文字替代）
4. 用 Write 工具保存到 `reports/[slug]_research_[YYYY].html`
5. 用 `SendUserFile` 推送给用户

## 质量自检

```
□ 报告结构与客户提纲一一对应？
□ 执行摘要每条判断有数据支撑或【分析预测】？
□ 所有关键数据标注了来源和日期？
□ 数据缺口已用【分析预测】补充，非留空？
□ 战略描述区分"已落地"和"规划中"？
□ Markdown 末尾有来源索引？
□ HTML 文件已生成并推送给用户？
□ HTML 包含 SVG 图表（趋势/拆分数据不得纯文字）？
□ 没有无信息量的模糊表述？
```
